2. 遍历切词列表    for i in range(len(word_list))
3. 含‘第一’字段的词的判定逻辑：if ‘第一’in word_list[i]
      （1）“第一”+量词，基本上是切成一整个词，if len（word）=3时，word.split(“”)[-1]把最后一个量词切出来，调用数据库量词表进行对比，in则返回非极限词，not in 则返回极限词
      （2）“第一”+形容词，基本上是将第一和下个形容词分开切，if len（word）=2时，检测后一个词的词性，如果是形容词，返回极限词
      （3）名词+名词+“第一”， 基本上会将名词和第一切成不同的词，if len（word）=2时，检测前一个词的词性，如果是名词，调用数据库名词表进行对比，in 则返回极限词，not in则返回疑似词，需要客户确认
4. 含‘最’字段的词的判定逻辑：if ‘最’in word_list[i]
      （1）“最X”一般切词会切成一个整词， if len（word）=2时，word.split(“”)[-1]把最后一个形容词切出来，调用数据库量词表进行对比，in则返回极限词，not in 则返回疑似词
      （2）“最XX”可能切成一个整词，也可能切成“最”，“XX”两个词， if len（word）=3时，word.split(“”，1)[-1]最后一个词调用数据库进行对比；if len（word）=1时，则将word_list[i+1]这个词调用数据库进行对比。
5. 其他词直接调用非常用极限词库进行比对；
6. 生成检测结论，结论生成内容及格式参考草图；

网上部分自动识别插件的局限性
网上流传着大量的类似上图的检测工具，
但是这些工具的局限性很明显，算法相对简陋，只能初步的识别出包含极限字眼的词，
但是在特定的语境下，包含极限字眼的词并不属于极限词，这就造成了需要用户自己人工再识别一次，无形间增加了许多工作量，
我们设计的算法中，根据一句话的语法构成，例如部分常用极限词的词性属于程度副词，程度副词+量词在大多数情况下是不属于极限词的范畴的，所以在我们的数据库的设计中，拥有一个专属的单字量词库和单字形容词库进行匹配识别。
